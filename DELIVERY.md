## 注意事项

### 1. 图片资源

项目中预留了图片资源目录，但未包含实际图片文件。需要根据设计稿准备以下图片：

- `images/base/attention-icon.png` - 注意力测试图标
- `images/base/dst-icon.png` - 数字广度测试图标
- `images/base/memory-icon.png` - 情景记忆测试图标
- `images/base/tmt-icon.png` - 连线测试图标
- `images/base/raven-icon.png` - 瑞文推理图标

如果没有图片，页面会使用渐变色背景作为占位符，不影响功能使用。

### 2. API 接口

项目中的 API 地址是示例地址，需要修改为实际的后端 API 地址。

后端需要实现以下接口：

- `POST /api/test/submit` - 测试数据上报接口
- `GET /api/game/config/{gameType}` - 游戏配置接口（可选）

接口规范详见 `DEPLOYMENT.md` 文档。

### 3. ES5 语法

整个项目严格使用 ES5 语法，不包含任何 ES6+ 特性（除了 Promise，因为大多数目标浏览器都支持）。

如果需要支持更老的浏览器，可以引入 Promise polyfill。

### 4. 吉祥物图片

PRD 文档中提到的"猴子 IP"吉祥物，目前使用 emoji 表情符号 🐵 作为占位符。

如果有实际的吉祥物图片，可以替换为：

```html
<div class="mascot">
    <img src="../images/base/mascot.png" alt="吉祥物">
</div>
```

### 5. 后续开发

其他4个游戏（DST、情景记忆、TMT、瑞文推理）的框架已预留，可以参考注意力测试游戏的实现方式进行开发。

详细的开发指南请参考 `DEVELOPMENT.md` 文档的"添加新游戏"章节。
